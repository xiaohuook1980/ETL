单价配置 v2 mock （组合关键字 + AND 匹配 + 时段）

设计要点：
1. 一个项目一份配置（项目级，不再是 unit_prices 那种跨项目独立行）
2. 项目级配 3 个匹配维度的列名（来自 xlsx 列映射或 extra_data，留空=该维度不启用）
3. 每条单价规则 = 3 列关键字组合（AND）+ 单价 + 生效时段；空/* = 通配
4. 多条命中按"具体度优先"（非通配字段越多越优先），同分按"上→下"顺序
5. 关键字内逗号分隔多候选（OR），如"员工,临时工"
6. 时段过滤：取 attendance.shift_date 在 [生效起, 生效止] 内的规则
项目配置
列名要在该项目的 attendance 字段或 extra_data 里能找到才有效

匹中规则：3 列 AND；每列内"逗号分隔关键字" OR；带 * 的列不参与匹配；时段不在 [生效起,生效止] 内的规则跳过。
具体度 = 非通配字段数；多匹中按具体度降序 → 同分按 # 升序。

注意点：

  • "无匹中"的 attendance 行 calc 时按 0 算（或抛预警让用户加规则）
  • 关键字支持子串模糊匹配（"白班" 命中"早白班/晚白班"）—— 待用户拍板
  • 项目级匹配维度可改但要小心：改完老规则的关键字含义可能错位
  • 这个 mock 只展示样式，未对接后端
