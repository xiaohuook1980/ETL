规则：
mart 字段 ← 文件列名（支持 * 通配；空=不映射） 说明
姓名 name_raw * ← 必填
应发工资 payable_amount * ← 必填；老希锐项目可填"应发"
是否替班 is_substitute ← 可选；列值非空 → is_substitute=1
替班姓名 substitute_name ← 可选；is_substitute=1 时本列值入库
备注 extra_data ← 未映射列全装 JSON
必填：姓名 + 应发工资。
工资表的列映射极简：mart.wage_sheets 只关心 name_raw / payable_amount / is_substitute / substitute_name； 扣项目（保险/个税/水电）全进 extra_data，calc 不读。
schema 缺口：is_substitute 这种"列值非空 → 标志位 1"的逻辑当前 column_mapping 表达不了，要么硬编码 standard handler 处理，要么扩 mapping 字段加"模式"。
