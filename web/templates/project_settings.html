发薪流水的业务日期判定
每条发薪记录归到哪个业务周期？两套规则配合：先指定，指定不到再推断。
① 指定规则 从字段里直接抽出业务日期，命中即用，不再推断
每条规则：文件特征列判断"这是哪种文件"，提取列指定从哪一列抽业务日期。
• 文件特征列：空格分隔列名，表头里必须按顺序连续相邻出现规则才生效；列名本身含空格用 #$% 替代；空 = 总是生效
  用法：从表头复制几个连续列名粘贴即可（如 账号 户名 金额）
• 提取列：内置 date 解析器扫，抽到 年+月+日（如 2026.2.5）→ 完整业务日；抽到 年+月（如 2026.2）→ 自然月业务周期可定，非自然月报错跳过；抽到 仅月 → 年用 pay_time 推断
• 所有规则都没生效或都没抽到 → 走"② 推断规则"
启用 文件特征列（空格分隔，按顺序连续相邻；空=总是生效） 提取列（| 分隔多候选） 最近命中
② 推断规则 从某列读 datetime 减偏移得业务日；指定规则没匹配时按此规则推
每条规则：文件特征列识别文件 + 推断列读 date - 偏移 = 业务日
• 文件特征列空 = 该规则适用所有文件（兜底）
• 推断列必填，必须是表头里能读出 date/datetime 的列（如 付款时间/经办日）
• 偏移 N=0 即"读出来直接当业务日"；N≥1 减后跨业务周期（如月结减 1 月）
• 都没匹配 → 最终 fallback：直接用 pay_time 当业务日
启用 文件特征列（空=所有文件） 推断列（| 分隔多候选） 偏移 N 单位 最近命中
