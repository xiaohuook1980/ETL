列映射 文件列 → mart.payrolls 字段
规则：
mart 字段←文件列名（支持 * 通配；逗号分隔多候选；空=不映射）说明
付款时间 pay_time * ← 必填；同义词：付款/转账/到账/经办；
无此列时填 $bill_month 用账单月份兜底
姓名 name_raw * ← 必填；同义词：姓名/户名/收款人
金额 work_amount * ← 必填；同义词：付款金额/实发金额/转账金额

业务日期判定 每条发薪记录归到哪个业务周期？两套规则配合：先指定，指定不到再推断
① 指定规则 从字段里直接抽出业务日期，命中即用，不再推断
每条规则：文件特征列判断"这是哪种文件"，提取列指定从哪一列抽业务日期。
文件特征列：空格分隔列名，按顺序连续相邻；列名本身含空格用 #$% 替代；空 = 总是生效
提取列：抽到 年+月+日 → 完整业务日；年+月 → 自然月业务周期可定；仅月 → 年用 pay_time 推断
加载中…
② 推断规则 从某列读 datetime 减偏移得业务日；指定规则没匹配时按此规则推
每条规则：文件特征列识别文件 + 推断列读 date - 偏移 = 业务日
文件特征列空 = 该规则适用所有文件（兜底）
推断列必填，必须是表头里能读出 date/datetime 的列（如 付款时间 / 经办日）
偏移 N=0 即"读出来直接当业务日"；N≥1 减后跨业务周期（如月结减 1 月）
都没匹配 → 最终 fallback：直接用 pay_time 当业务日
加载中…
③ 账单月份兜底 无付款时间列时启用，从 mini_a_bill.bill_month 取月份归到该月业务周期
启用条件：列映射里 pay_time 配置为 $bill_month 时本规则才生效
行为：pay_time / parsed_shift_date 填账单月份首日 00:00:00；business_month 直接归该月；payroll_kind 标为 bill_month_only
典型场景：达达 / 京东系跑腿平台导出的转账计划回执，无时间字段
跟 ① ② 关系：① 指定 / ② 推断 都没匹配 → 看本规则是否启用 → 启用则用 bill_month；否则最终 fallback 到 pay_time
